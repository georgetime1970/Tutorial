# 泛型,trait,生命周期

## 泛型

- 类似函数的参数,这是一种类型参数
- 与TS不同的是,rust必须为泛型指定trait,也就是指定泛型的能力,否则无法使用参数的方法(能力)

```rust
fn largest<T>(list: &[T]) -> &T {}
```

这里的T指的就是函数参数的类型,可以是任意类型,返回值是类型为T的值的引用

```rust
struct Point<T> {
    x: T,
    y: T,
}

impl<T> Point<T> {
    fn x(&self) -> &T {
        &self.x
    }
}
// 任意Point<T>实例都实现x方法
// 注意必须在 impl 后面声明 T，这样就可以在 Point<T> 上实现的方法中使用 T 了。
// 通过在 impl 之后声明泛型 T，Rust 就知道 Point 的尖括号中的类型是泛型而不是具体类型


impl<T> Point<T>
where T: std::fmt::Display {
    fn print(&self) {
        println!("{}", self.x);
    }
}
// 只有当 T 实现了 Display，Point<T> 才拥有 print 方法


impl Point<f32> {
    fn distance_from_origin(&self) -> f32 {
        (self.x.powi(2) + self.y.powi(2)).sqrt()
    }
}
// 这段代码意味着 Point<f32> 实例会有一个方法 distance_from_origin
// 而其他 T 不是 f32类型的 Point<T> 实例则没有此方法

fn main() {
    let p = Point { x: 5, y: 10 };
    println!("p.x = {}", p.x());
}
```

## Trait

- 通过 trait 以一种抽象的方式定义共同行为
- 注意: 是用于定义行为的,是抽象的也可以有默认行为

### 定义 trait

- 使用 `trait` 关键字
- `trait`和`impl`实现一样,里面函数的第一个参数是`&self`, 指代对实例的引用

```rust
trait DoSomething {
    // 抽象方法
    fn run(&self);
    // 默认方法
    fn sleep(&self) {
        dbg!("I am sleeping");
    }
}
```

### 实现trait

```rust
struct Person {
    name: String,
    age: u8,
}

impl DoSomething for Person {
    // 实现抽象方法
    fn run(&self) {
        dbg!("I am running");
    }
    // 重写默认方法
    fn sleep(&self) {
        dbg!("I am eating");
    }
}
```

**孤儿规则**

如果你想给一个类型实现一个接口，那么这两者中至少有一个必须是你自己定义的。

- ✅ 可行：给自己的 Struct 实现 系统的 Trait（比如给 Person 实现 Display）。
- ✅ 可行：给系统的类型 实现 自己的 Trait（比如给 i32 实现你的 DoSomething）。
- ❌ 禁止：给系统的类型 实现 系统的 Trait（比如你不能给 String 实现 Display，因为这两者都是 Rust 官方定义的）。

### trait 作为参数

- trait参数的目的就是给泛型指定能力范围
- Trait 作为参数就像是一张 “入场券”
- 你必须具备指定trait的能力才能作为这个函数的参数,否则函数不知道你能做什么

1. impl Trait 语法（最常用、最简洁）
   这是最易读的方式，适用于简单的函数参数。它告诉编译器：这个参数必须是实现了该 Trait 的某种类型。

```rust
// 只要实现了 DoSomething 和std::fmt::Display 的类型都能传进来
fn execute(item: &(impl DoSomething  + std::fmt::Display)) {
    item.run();
}

let san = Person { name: "张三".into(), age: 18 }; // into()利用上下文推导,进行类型转换
execute(&san);
```

2. Trait Bound（限定语法）
   这是 impl Trait 的完整写法。当你需要多个参数是同一种类型，或者逻辑复杂时，必须使用它。

```rust
// 强制要求 arg1 和 arg2 的类型必须完全一致，且都实现 DoSomething
fn combine<T: DoSomething>(arg1: T, arg2: T) {
    arg1.run();
    arg2.run();
}

let san = Person { name: "张三".into(), age: 18 };
let si = Person { name: "李四".into(), age: 20 };
combine(san, si);
```

3. where 子句（最整洁）

- 约束泛型必须具备某种能力

```rust
// 需求：T 必须能运行，且必须能打印输出
fn complex_task<T>(item: T)
where
    T: DoSomething + std::fmt::Display
{
    item.run();
    println!("Status: {}", item);
}
```

### 返回实现了 trait 的类型

```rust
// 返回一个实现了 DoSomething的结构体
// Person必须已经提前实现了DoSomething trait才行
fn new_perosn() -> impl DoSomething {
    Person {
        name: "王五".into(),
        age: 19,
    }
}

fn main(){
  let wu = new_perosn();  // 调用函数
  wu.run(); // 返回的struct自动实现了run方法
}
```

### 综合例子

```rust
// 定义一个struct
struct Bird {
    name: String,
}

// 定义一个trait
trait Fly {
    fn fly(&self);
}

// 为结构体实现trait
impl Fly for Bird {
    fn fly(&self) {
        dbg!(&self.name, "正在飞---trait方法");
    }
}

// Bird 独有的方法
impl Bird {
    fn play(&self) {
        dbg!(&self.name, "喜欢玩足球---固有方法");
    }
}

// 一个需要Fly能力的函数
fn can_fly<T: Fly>(name: &T) {
    dbg!("我拥有飞的能力---泛型函数");
    name.fly();
}

fn main() {
    // 实例化结构体
    let san = Bird {
        name: "zhangSan".into(),
    };

    // 调用san的固有方法
    san.play();
    // 调用san的trait方法
    san.fly();
    // 调用泛型函数
    can_fly(&san);
}
```

## trait 思考

- `trait`就是一个能力证明器,也是复用方法的集合,用在泛型函数中时也是一种约束(比如Fn,FnMut,FnOnce等闭包相关的trait)
- 主要用在泛型函数中指定参数的行为,只有指定了行为参数才能调用相应的方法
- rust的设计非常的严谨,比如`struct`如果自己`impl Struct` 实现了某种方法,在进入泛型函数时也不会认可这种方法,必须`impl Trait`再次实现/转发这种方法
- `trait`就像是一个证件一样,证明你会某种东西,而`struct`自己的`impl Struct`的固有方法则是你会但你没有证,也无法证明你会

**如何合理使用`impl Trait`和`impl Struct `**

- `impl Struct` 私有逻辑
- `impl Trait` 对外社交 和 通用算法

1. 第一步：先写 `struct` 和 `impl Struct`。把功能实现出来，让自己能跑通。
2. 第二步：当你发现有 2 个以上 的结构体有相同操作，或者有个 第三方函数（比如 `Iter` 或 `Serialize`）在管你要“证”时。
3. 第三步：把那部分重复的逻辑提取成 `Trait`，然后在 `impl Trait for Struct` 里调用你之前在第一步写好的固有方法。

## 额外补充

### Trait 对象（`dyn Trait`）

Trait 对象是 Rust 中实现**运行时多态**的方式。与泛型的**编译时静态分发**不同，Trait 对象在**运行时动态分发**。

- 语法：`Box<dyn Trait>` 或 `&dyn Trait`
- 本质：一个**胖指针**，包含数据指针 + 虚函数表（vtable）指针
- 使用场景：需要在同一集合中存入**不同类型**但实现了同一 Trait 的对象时

```rust
trait Draw {
    fn draw(&self);
}

struct Circle;
struct Square;

impl Draw for Circle {
    fn draw(&self) { println!("画圆"); }
}
impl Draw for Square {
    fn draw(&self) { println!("画方块"); }
}

// 同一个 Vec 里存放不同类型
fn draw_all(shapes: &[Box<dyn Draw>]) {
    for shape in shapes {
        shape.draw(); // 运行时动态分发
    }
}

fn main() {
    let shapes: Vec<Box<dyn Draw>> = vec![
        Box::new(Circle),
        Box::new(Square),
    ];
    draw_all(&shapes);
}
```

### `dyn Trait` vs 泛型 `T: Trait` 对比

| 特性     | 泛型 `T: Trait`（静态分发）  | Trait 对象 `dyn Trait`（动态分发） |
| -------- | ---------------------------- | ---------------------------------- |
| 分发时机 | 编译时（零开销抽象）         | 运行时（vtable 查表，有轻微开销）  |
| 同一集合 | 只能放**同一类型**的元素     | 可以放**不同类型**的元素           |
| 代码膨胀 | 编译器为每种类型生成专属代码 | 所有类型共用同一套代码             |
| 使用限制 | 无额外限制                   | Trait 必须满足**对象安全**规则     |

> **对象安全（Object Safety）**：Trait 的方法不能返回 `Self`，不能含泛型参数，否则无法构建 vtable。

### 静态 vs 动态分发写法对比

```rust
// 静态分发：编译期单态化，性能最优
fn print_x2<T: std::fmt::Display>(x: T) {
    println!("{x} {x}");
}

// 动态分发：运行时多态，可接受任意实现了该 Trait 的类型
fn print_x1(x: &dyn std::fmt::Display) {
    println!("{x}");
}
```
