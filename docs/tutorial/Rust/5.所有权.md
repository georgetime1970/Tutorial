# 所有权和切片

## 所有权规则

- 每个值都有一个变量，这个变量是该值的所有者
- 每个值在任意时刻有且只能有一个所有者
- 当所有者超出作用域（scope）时，该值将被删除(调用drop并回收内存)
- 所有权可以转移(move)
- 赋值/传参/函数返回时,如果类型未实现Copy,所有权将发生转移
- 基本数据类型(如整数/布尔/浮点/不可变引用&T)实现了Copy,会自动复制,不会转移所有权,设计原因和性能有关

## 移动 move 和复制 copy

1. 基本数据类型（存储在stack上）赋值给其他变量不会转移所有权,因为实现了Copy;

```rust
let a = 1;
let b = a;  // 直接进行了复制操作
println!("{a}, {b}"); // 不会出现错误
```

2. 复杂数据类型（存储在heap上），直接赋值给变量会转移所有权.

```rust
let x = String::from("hello");
let y = x;  // 浅拷贝+x失效，x此时移动所有权到y; 修复：1.let y = x.clone() 或者 2.let y = &x
println!("{x}, {y}"); // 报错：x value borrowed here after move
```

`let y = x` 这一步是将stack里的指针（ptr），数据长度（len），数据总大小（capacity）这些信息复制给y（浅拷贝）,并且将x回收，避免最后二次释放同一个指针的数据（二次释放）;
本质是在没有垃圾回收器的情况下，实现垃圾回收的一种解决方案，保证内存安全

3. 函数传参，同样遵循前所有权规则，不是复制就是移动

4. 函数返回值，同样会发生所有权的转移

## 克隆: clone()

主要针对heap数据的深拷贝(heap生成2份相同的数据,stack上2份不同的指针指向自己的heap)，数据较大时比较消耗资源

> 所以基本数据类型自动实现了Copy,因为对性能影响较小,但本质也遵循所有权规则,只是编译器自动进行了复制
>
> 复制这些微小数据(基本数据类型)的开销甚至比处理它们的指针引用还要低，因此编译器默认开启自动复制

## 引用(Reference): &

**是一个语法概念,指的是引用某些值而不获得所有权**

`let y = &x`: y创建了一个指针指向x,x里面是指向的heap的数据

> 本质是复制了stack上的数据,也就是指针数据

## 借用

**是使用引用的一种行为,指的是通过引用来访问数据而不获取所有权的过程,当你创建了一个引用,就是在借用数据**

- `&`：不可变引用： 不可修改变量的内容
- `&mut`: 可变引用： 可以修改变量的内容,借用周期中不可再使用原变量(独占引用)

规则：同作用域同变量时，只能满足下列条件之一

1. 一个可变引用，防止数据竞争
2. 无限个不可变引用

## 解引用 \*

- 解引用后可以读取到真实的原数据

## 悬空引用

引用一个指针指向一个已被释放或已重分配的内存

## 切片 slice

**不持有所有权的一种数据类型**
**指向字符串中一部分数据的引用**

语法： [开始索引..结束索引]

- 本质是一种借用行为

```rust
fn main() {
    let s = String::from("hello world");
    let hello = &s[..5];  // [..5] = [0..5]
    let world = &s[6..];  // [6..] = [6..s.len()]
    let hello_world = &s[..]  // [..] = [0..s.len()]
    println!("{hello} {world} {hello_world}")
}
```

> 极其不推荐直接使用 `&s[start..end]` 来对字符串（String 或 &str）进行切片，除非你百分之百确定索引位置
> 在 TS 或 Python 中，`s[0..2]` 通常指“前两个字符”。但在 Rust 中，切片索引指的是字节（Bytes）
> 简单点说就是你可能切不准会导致panic,推荐使用`.chars()` 迭代器

在 Rust 中，切片（Slice） 是一种不持有所有权的特殊类型，它允许你引用集合中一段连续的内存序列。

1. 什么是切片？
   切片在内存中被称为胖指针（Fat Pointer）。它由两部分组成：
   - 指针：指向数据序列中起始元素的内存地址。(只有这个就是瘦指针)
   - 长度：记录该切片包含多少个元素。

   这种结构让切片非常高效，因为它只是对现有数据的一个“窗口”视图，不需要复制底层数据。

2. 哪些数据可以切片？
   只要数据在内存中是连续存储的，就可以创建切片：
   - 数组 (Array)：如 `[i32; 5]`，切片类型为 `&[i32]`。
   - 动态数组 (Vector)：如 `Vec<T>`，切片类型为 `&[T]`。
   - 字符串 (String)：切片类型为 `&str`。

3. &str 与切片的关系
   &str 本质上就是一种特殊的切片。
   - 定义：&str 是对 String 或字符串字面量中一段 UTF-8 格式字节序列的引用。
   - 与 String 的关系：String 是拥有所有权、可增长的堆内存字符串；而 &str 是指向 String（或静态内存）某一部分的视图。
   - 通用性：在编写函数参数时，推荐使用 &str 而不是 &String，因为 &str 既能接收 &String（通过自动强制多态），也能接收字符串字面量和 String 的局部切片。
4. **只能对引用进行切片**
   - 切片不拥有所有权
     在 Rust 中，切片类型（如 [i32] 或 str）是所谓的 DST（动态大小类型）。这类类型的大小在编译时是未知的。
     如果你不使用引用（如直接写 let s: [i32] = [1,2,3][..2]），编译器不知道该在栈上为你分配多少空间。
     通过使用引用（&[T]），切片变成了固定大小（指针+长度），编译器就能轻松处理它。
     所以，你看到的 &[T] 或 &str 实际上是“对一个连续序列的借用”。

5. 关键语法示例
   ```rust
    let arr = [1, 2, 3, 4, 5];
    let slice: &[i32] = &arr[1..4]; // 包含索引 1, 2, 3 的元素
    let s = String::from("Hello Rust");
    let word: &str = &s[0..5]; // "Hello" 部分的切片
   ```

需要注意的是：对字符串切片时，索引必须落在合法的 UTF-8 字符边界上，否则程序会发生崩溃（Panic）。
