# 异步编程基础：Async、Await、Future 与 Stream

## 1.异步编程简介

Rust的异步编程能够让你以一种高效且安全的方式处理 I/O 密集型或高并发任务，如 Web 服务、网络通信等。在 Rust 中，异步靠`async`/`await`语法、`Future`和`Stream`特性实现。

## 2.async 和 await 关键字

- `Future`: 是一个现在也许还没准备好，但会在将来某个时刻准备好的值,实现了 `Future` trait 的类型。(JS中的Promise)
- `async`：用于声明异步的代码块、函数或闭包，其返回一个实现了`Future` trait的类型。
- `await`：用于等待一个`Future`完成并获取其结果。

### 异步函数示例

```rust
async fn say_hello() {
    println!("Hello from async!");
}
```

调用`async fn`会立即返回一个`Future`，实际的执行要靠 runtime 调度，可以通过`.await`获得最终结果。

### 简单的 Future 示例

```rust
async fn learn_async() -> u32 {
    7
}

async fn async_main() {
    let value = learn_async().await;
    println!("value = {}", value);
}
```

## 3.Future Trait

`Future`是一个代表还未完成但最终会产生某个值的值的 trait。标准库定义如下：

```rust
pub trait Future {
    type Output;
    fn poll(
        self: Pin<&mut Self>,
        cx: &mut Context<'_>
    ) -> Poll<Self::Output>;
}
```

正常开发无需手动实现`Future`，编译器自动帮我们生成。

## 4.运行异步代码：异步运行时

标准库本身不包含异步运行时，因此需要引入第三方库如`tokio`或`async-std`。常用的是[Tokio](https://tokio.rs/)。

**Cargo.toml：**

```toml
[dependencies]
tokio = { version = "1", features = ["full"] }
```

**main.rs 示例：**

```rust
#[tokio::main]
async fn main() {
    async_main().await;
}

async fn async_main() {
    println!("Hello from async world!");
}
```

`#[tokio::main]`宏会自动创建并启动 Tokio runtime。

## 5.使用async-std

另一常用选择：

**Cargo.toml:**

```toml
[dependencies]
async-std = "1"
```

**main.rs:**

```rust
#[async_std::main]
async fn main() {
    async_main().await;
}
```

## 6.Stream：异步流

`Stream`可理解为异步版的 Iterator。Tokio 和 futures 提供支持。

**引入依赖用于示例：**

```toml
[dependencies]
futures = "0.3"
tokio = { version = "1", features = ["full"] }
```

**Stream 示例：**

```rust
use futures::stream::{self, StreamExt};

#[tokio::main]
async fn main() {
    let mut stream = stream::iter(vec![1, 2, 3, 4]);

    while let Some(item) = stream.next().await {
        println!("Stream item: {}", item);
    }
}
```

## 7.常见应用：异步HTTP请求

以`reqwest`库为例：

**Cargo.toml:**

```toml
[dependencies]
tokio = { version = "1", features = ["full"] }
reqwest = { version = "0.11", features = ["json"] }
```

**main.rs:**

```rust
#[tokio::main]
async fn main() -> Result<(), reqwest::Error> {
    let resp = reqwest::get("https://www.rust-lang.org")
        .await?
        .text()
        .await?;
    println!("Body: {}", &resp[0..20]);
    Ok(())
}
```

---

## 总结

Rust 的异步编程强大且高效，建议优先掌握 async/await 基本用法，再结合 tokio 或 async-std 等运行时进行实际开发。Stream 适用于异步序列场景。

- `async fn` 或 `async` 块返回 Future，需要通过 `.await` 执行。
- 需配合异步运行时使用（如 tokio/async-std）。
- Stream 对应异步迭代流程。
- 实际编程以 async/await 为主，配合生态库。

更多信息参考：[Rust 异步官方文档](https://rust-lang.github.io/async-book/zh/index.html)
